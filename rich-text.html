Rich text formatting

Format content before it reaches the target.

Create or paste formatted text, review a plain-text fallback, and choose an output method that suits the destination.

BIU• List14 ptClear

Output choices

Plain typing or formatted paste.

Use character-by-character typing for broad compatibility, or rich paste when the destination accepts HTML clipboard content.

Plain Text Typing

Uses the existing typing engine, including speed, human-like timing, Mistake Mode, countdown, pause, resume, and progress.

Works almost everywhere.

Rich HTML Paste

Places HTML and plain text on the Windows clipboard, then pastes into the selected app.

Formatting depends on target-app support.

i

Plain Text Typing works almost everywhere. Rich HTML Paste keeps formatting in apps that support rich paste.

Editor actions

Choose the form you need.

The editor keeps formatted content and its plain-text version available without pretending every target app behaves the same way.

  • Copy Plain Text for unformatted clipboard content
  • Copy HTML for the generated HTML source
  • Copy Rich Format for HTML plus plain-text clipboard formats
  • Use for Typing to send the plain-text fallback to Auto Typer
  • Use for Rich Paste to prepare formatted clipboard output

Safe HTML handling

Focused on text formatting.

Imported content supports common text structure and ignores unsafe or unsupported elements. Scripts are not executed and external resources are not loaded automatically.

  • Paragraphs, line breaks, emphasis, headings, lists, and simple links
  • Plain-text fallback generated from the document
  • Undo, redo, copy, paste, select all, and clear formatting
  • No JavaScript execution from imported HTML
!

If formatting does not appear, switch to Plain Text Typing.
The receiving app decides which clipboard format it accepts.
