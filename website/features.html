Product features

Useful automation, without the clutter.

Choose the tool you need, set the timing, and keep start, pause, resume, stop, and progress controls in view.

The complete toolset

Each feature has a clear purpose. Core typing, clicking, formatting, and playback tools remain available offline.

Auto Typer

Type prepared text into another Windows app with adjustable speed, countdown, pause, resume, stop, and progress tracking.

Human-like Typing

Add controlled timing variation, word pauses, and sentence pauses so output feels less mechanically uniform.

Mistake Mode

Optionally introduce small letter or word mistakes and, when enabled, correct them during plain-text typing.

Rich HTML Paste

Prepare formatted content and place both HTML and plain-text formats on the clipboard for compatible target apps.

Auto Clicker

Send left, right, or middle clicks with interval, action, location, countdown, pause, resume, and stop controls.

Task Recorder

Capture short supported action sequences and replay them using speed, repeat, continuous-loop, pause, resume, and stop options.

Optional AI Writing

Generate draft text with your own OpenAI API key. It is optional, requires internet, and does not affect offline typing or clicking.

See the official Privacy Policy before using connected features.

Offline-friendly Core

Auto Typer, Auto Clicker, rich text editing, Task Recorder playback, themes, and local settings do not need an internet connection.

WinUI 3 Interface

Responsive navigation, Segoe UI typography, rounded controls, light and dark themes, and scaling designed for modern Windows displays.

Settings and Workflow

Save theme and automation preferences, minimise on start, restore after finish or stop, and keep status feedback visible.

One active workflow

Clear state across each tool.

Autotyper Pro keeps automation jobs separate so typing, clicking, and task playback do not compete for input.

  • Friendly validation before a job begins
  • Consistent Ready, Starting, Running, Paused, Stopped, Finished, and Error states
  • Stop remains available during active automation
  • Closing the app safely ends active work
